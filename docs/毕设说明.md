# 毕设说明文档

## 项目名称

**智能文件/书签管理系统** — Smart File & Bookmark Manager

## 选题背景

随着互联网信息的爆炸式增长，个人数字资产管理成为痛点：
- 文件散落在各个文件夹，难以快速定位
- 浏览器书签越积越多，层次混乱，缺乏分类
- 缺乏统一的知识管理平台

本系统旨在构建一个集文件管理与书签管理于一体的智能平台，帮助用户高效组织和检索个人数字资源。

## 系统架构

```
┌─────────────────────────────────────────┐
│              前端层 (Vue3)              │
│   Element Plus UI / Pinia / Axios       │
└──────────────┬──────────────────────────┘
               │ HTTP REST API (JWT 认证)
┌──────────────▼──────────────────────────┐
│              后端层 (Flask)             │
│   Python / SQLAlchemy / Flask-JWT       │
└──────────────┬──────────────────────────┘
               │ SQLAlchemy ORM
┌──────────────▼──────────────────────────┐
│            数据层 (MySQL)               │
│   users / files / bookmarks / tags      │
└─────────────────────────────────────────┘
```

## 核心功能模块

### 1. 用户认证模块
- JWT 双令牌机制（Access Token + Refresh Token）
- 注册时自动创建默认标签
- 密码哈希存储（Werkzeug）

### 2. 文件管理模块
- 多格式文件上传（PDF/DOC/XLS/PPT/图片/代码/压缩包等30+格式）
- 分页查询、关键词搜索、类型筛选、标签筛选
- 文件下载计数、浏览计数
- 收藏标记

### 3. 书签管理模块
- 单条添加 + 浏览器书签HTML批量导入
- 智能分类建议（根据域名和标题关键词）
- URL格式验证、域名自动提取
- 分类管理（未分类/技术学习/影音娱乐/购物/社交等）

### 4. 标签系统
- 全局标签池，文件和书签共用
- 10种预设颜色，支持自定义
- 多对多关联关系
- 热门标签排行统计

### 5. 数据面板
- 文件总数、书签总数、标签总数、存储空间
- 文件类型分布图
- 近7天新增统计

## 数据库设计

### E-R 图

```
┌──────────┐    1:N    ┌──────────┐    N:M    ┌──────────┐
│   users  │◄──────────┤  files   │◄─────────►│   tags   │
└──────────┘           └──────────┘  file_tags └──────────┘
     │                                         ▲
     │ 1:N      ┌───────────┐    N:M          │
     └─────────►│ bookmarks │◄────────────────┘
                └───────────┘  bookmark_tags
```

### 核心表结构

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| users | 用户表 | username, email, password_hash |
| files | 文件表 | filename, original_name, file_size, file_type |
| bookmarks | 书签表 | title, url, category, visit_count |
| tags | 标签表 | name, color |
| file_tags | 文件-标签关联表 | file_id, tag_id |
| bookmark_tags | 书签-标签关联表 | bookmark_id, tag_id |

## 技术亮点

1. **RESTful API 设计** — 标准的资源路径 + HTTP 方法语义
2. **JWT 双令牌认证** — 安全性更好，Access Token 短期 + Refresh Token 长期
3. **智能分类算法** — 基于域名规则库 + 标题关键词匹配的书签自动分类
4. **AI 标签建议（扩展）** — 基于 TF-IDF 关键词提取的智能标签推荐
5. **前端组件化** — Vue3 Composition API + Element Plus 组件库
6. **API 代理** — Vite 开发服务器代理解决跨域问题
7. **响应式布局** — 侧边栏导航 + 内容区自适应
